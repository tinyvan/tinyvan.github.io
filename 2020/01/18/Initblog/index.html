Hexo的艰难历程

2020年01月18日 366 字 大概 1 分钟 Web Hexo 折腾

去年在Github探索中发现了Hexo库,打断了我徒手(手写html)的想法

为凸显个性(粉红的少女心)~~,决定以Nexmoe作为模板

ubuntu

sudo apt install git nodejs npm
sudo npm install hexo-cli -g
hexo init 'Blog'
cd Blog/themes/
git clone https://github.com/nexmoe/hexo-theme-nexmoe.git nexmoe
cd nexmoe
npm install hexo-wordcount --save
cp -i _config.example.yml _config.yml
hexo s --debug
# 此过程不完善,详情以下文为主

安装hexo模块过程中发生了一些错误:

  • npm全局安装没有写权限

    解决:sudo

  • 报错npm ERR! cb()never called!

    解决:博客园

服务运行中发生了一些错误:

  • 无论图片放在哪里,都无法显示(与文章(即.md)无关的图片)

    解决:删库跑路重装,在nexmoe(主题文件夹)/source/创建images文件夹,图片扔进去,并修改主题配置文件的图片路径

  • markdown插入代码段显示有误

    解决:无正规套路,如下可修正

    • 站点配置文件修改

      highlight:
    enable: false #true改false

    • 执行hexo clean

    • 删除package.json的marked项

    • 把删除的marked补回去

      以上每项做完后都启动hexo s --debug测试一下

      除了第一项外都可打乱顺序,如果无效多试几次

      也可以发表评论求助,我乐于帮助每一位有困难的人

小提示:

  • 在站点根目录和主题目录均有source文件夹,前者为文章服务,后者受命于主题
